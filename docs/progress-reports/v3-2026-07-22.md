## 1. Strict / fail-closed ratchet

Command: `./scripts/test-fail-closed.sh` (sets `HERMIT_FAIL_CLOSED=1`; ptrace
backend; log level default; relaxations: none — this is the strictest mode).

The script is fail-fast (`set -euo pipefail`) and **aborted at the fourth
enabled test**, so it did not reach the remaining ~19 integration targets and
did not print its final ratchet summary line. Exact results before abort:

| Test | Result |
| --- | --- |
| `arbitrary_binaries/arbitrary_binary_commands_are_bounded` | PASS |
| `arbitrary_binaries/arbitrary_binary_lists_are_curated_for_ci` | PASS |
| `clock_determinism/clock_apis_are_deterministic_across_five_runs` | PASS |
| `clock_determinism/strict_mode_eliminates_native_clock_nondeterminism` | **FAIL** → abort |

**Net: 3 passed, 1 failed, then aborted.**

Failure root cause (verbatim from `/tmp/progress-strict.log`): guest
`/bin/date +%s%N` issued a `clock_gettime` that Detcore's fail-closed policy
rejected:

```
thread 'main' (1) panicked at detcore/src/lib.rs:1183:21:
unsupported syscall: ClockGettime(ClockGettime { raw: SyscallArgs { arg0: 0, ... } })
thread 'main' (1) panicked at .../core/src/panicking.rs:225:5:
panic in a function that cannot unwind
  19: reverie_process::clone::clone_with_stack::callback (reverie-process/src/clone.rs:27)
Error: Sandbox container exited unexpectedly
     > Process exited with code: Signaled(SIGSEGV, true)
```

This test is **not** listed in
`hermit-cli/tests/fail_closed_known_failures.tsv`, so it is an unexpected
failure on this host. Because the abort occurred in the fail-closed path only,
and because this host cannot fault CPUID (see host limitations), the most
likely cause is a raw `clock_gettime` reaching the fail-closed policy where a
CPUID-faulting host would route it through the vDSO. It cannot be conclusively
classified as a product regression versus a host limitation without re-running
on a host with CPUID faulting. Note the same `date` guest passes cleanly in
normal strict mode (see the working-envelope L1–L4 results below).

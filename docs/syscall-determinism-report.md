# Hermit syscall determinism report

> Generated by `scripts/determinism_report.py`. Do not edit by hand; re-run the tool to refresh.

This report audits which Linux syscalls Detcore intercepts, how each is handled for determinism, and which syscalls real applications actually issue. Use it to prioritise determinism-envelope expansion: the **Gaps** section lists syscalls that target apps need but Hermit does not handle.

## Classification method

Classifications are parsed from the dispatch `match` in `detcore/src/lib.rs`:

| Category | Meaning |
| --- | --- |
| `FULL` | Dedicated `handle_*` handler, unconditional: Detcore emulates/sanitizes the call. |
| `PARTIAL` | Handled but source-flagged as not fully deterministic, or gated on a config flag (e.g. `--virtualize-time`). |
| `PASSTHROUGH` | Forwarded to the host via `passthrough()`; deterministic only if the underlying operation is. |
| `MISSING` | Not in dispatch (returns `ENOSYS` unless `--allow-passthrough`), or an explicit `ENOSYS`/`panic!` stub. |

## Reproduction

Regenerate this report with::

```sh
# 1. Capture per-app syscall profiles (strace -f -c prints distinct
#    syscalls with call counts). Representative short runs:
strace -f -c -o redis.strace \
  redis-server --port 7799 --save '' --appendonly no --daemonize no  # ^C after startup
strace -f -c -o nginx.strace nginx -c /path/to/minimal.conf -p /tmp   # daemon off
strace -f -c -o py.strace python3 -c "import json,socket,os,threading,time; ..."
strace -f -c -o go.strace ./a_small_go_binary

# 2. Cross-reference against the Detcore dispatch and emit this report:
scripts/determinism_report.py --lib detcore/src/lib.rs \
  --strace redis=redis.strace --strace nginx=nginx.strace \
  --strace python=py.strace --strace go=go.strace \
  --out docs/syscall-determinism-report.md
```
The syscall *sets* are host- and version-dependent (libc, kernel, app build); call *counts* vary run to run. The Detcore classifications come directly from `detcore/src/lib.rs` and are stable for a given checkout.

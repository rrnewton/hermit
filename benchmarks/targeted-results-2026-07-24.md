# Targeted backend performance results

Date: 2026-07-24 UTC
Hermit revision: `dd60278f31e677fcfb13a1e84b6537c3cc209722`
Host: AMD EPYC 9D85, 316 logical CPUs, Linux 6.17.13
`perf_event_paranoid`: 1

## Method

The command below built release Hermit plus the four C fixtures, ran one warmup,
then retained five wall-clock samples for every workload/backend pair:

```sh
PYTHONDONTWRITEBYTECODE=1 with-proxy ./benchmarks/targeted.py
```

Hermit rows used explicit `--strict`, `--log=error`, and no relaxation flags.
Every row passed a precheck requiring exit zero and byte-identical stdout to
native. The runner rotated backend order between measured iterations and reports
the median. This is repeated L1 execution, not L2 verification or L4 stress.

| --- | ---: | ---: | ---: | ---: |
| CPU: 1M arithmetic iterations | 4.022 ms | 15.829 ms (3.94x) | 164.177 ms (40.82x) | 64.264 ms (15.98x) |

## Overhead model

**Ptrace matches syscall-linear overhead.** Removing the 15.8 ms fixed row cost,
the 100K-syscall row adds about 37.0 microseconds per call. The 10K mixed row
adds about 39.9 microseconds per call. Increasing executable text from about
0.4 KiB to 4 MiB leaves the ptrace median effectively unchanged (15.83 versus
15.97 ms).

**DBI matches translated-code startup cost and lowers syscall interception
cost.** Its small CPU workload establishes a roughly 164 ms fixed startup.
Traversing 4 MiB of text raises the median to 3.519 seconds while native remains
about 4 ms, directly exposing DynamoRIO translation cost. DBI is 2.54x faster
than ptrace on the 100K-syscall median and 1.93x faster on mixed work. It is not
the fastest choice for tiny or code-wide short-lived programs because startup
dominates.

**KVM matches fixed startup plus trapped-syscall cost.** CPU and large-code
medians are both about 64.3 ms. Above that fixed cost, both syscall workloads
add about 15 microseconds per call: 1.502 seconds for 100K calls and 150 ms for
10K calls.

## Caveats

This is a busy shared host. Load averages recorded at completion were
40.55/83.95/90.41 across 316 logical CPUs. Most rows were tight, but DBI's
100K-syscall samples ranged from 264.7 ms to 2.768 seconds and one DBI CPU sample
was 2.367 seconds; the required median includes that variance. These results
compare end-to-end process wall time, including each backend's setup and
teardown. Raw samples, exact commands, host metadata, and fixture sizes are in
the ignored `benchmarks/results/targeted/results.json` generated by the run.
